Tamsi Framework Architecture

Overview

Tamsi is a PHP Model-View-Controller (MVC) framework designed around a layered inheritance architecture. It provides a core framework (tamsi/) that is extended by an application layer (app/), and an auto-generated base layer (usr_generated/) that is extended by a user-customization layer (usr/). A code generator in util/ introspects database tables and produces full MVC scaffolding including administrative CRUD interfaces.

Architecture Layers

Config

Configuration

Purpose: Centralized configuration for the entire application.

  • Paths — include paths, URL base paths
  • Database — connection credentials and DSN
  • Routing — URL-to-controller mapping rules (routes.cfg.php)
  • Security — CORS, CSP, authentication settings
  • Constants — OAuth keys (Google, Facebook), app constants
  • Supports machine-specific overrides (*.cfg.php.home, *.cfg.php.gahum)

tamsi/  Core Framework

Purpose: Foundation MVC classes that define the framework contract.

  • No modifications allowed — shipped as a separate submodule
  • Defines ModelBase, ViewBase, CoreController
  • Provides TamsiModel, TamsiView, TamsiController extension points
  • Includes pagination (PaginatorView), file handling, security models

app/  Application Layer

Purpose: Extends/overrides tamsi/ classes. User-customizable.

  • Model extends TamsiModel
  • View extends TamsiView
  • Controller extends TamsiController
  • Allows override of core behavior without touching tamsi/
  • Contains app-specific models (Page, Role, Email, etc.)

usr_generated/  Auto-Generated Base

Purpose: Auto-generated base classes per database table.

  • Generated by util/generateMVC from database schema
  • Do NOT edit — regenerated whenever schema changes
  • Produces *Protected.php / *Public.php model pairs
  • Produces CRUD view templates (edit_*.tamsi.php, list_*.tamsi.php, new_*.tamsi.php)
  • Produces admin CRUD controllers

usr/  User Customization

Purpose: Extends usr_generated/ classes with custom logic.

  • [Model] extends *Protected or *Public
  • [Model]View extends *ViewBase
  • [Model]Controller extends auto-generated *Controller
  • User places custom business logic here
  • Survives regeneration — generated classes can be safely re-created

util/  Code Generator

Purpose: Automates MVC scaffolding from database tables.

  • generateMVC — shell script that calls all three generators
  • generateModel.php — inspects DB table schema, generates *Protected + *Public models
  • generateView.php — generates CRUD view templates with Bootstrap UI
  • generateController.php — generates admin CRUD controller with validation
  • Also generates administrative CRUD interfaces automatically

public_html/  Web Root

Purpose: Application entry point and static assets.

  • index.php — forces HTTPS, bootstraps the Tamsi framework
  • CSS, JavaScript, images, fonts
  • Frontend libraries: Bootstrap 4, jQuery, Font Awesome, Owl Carousel, etc.
  • Admin directory for back-office interfaces

Inheritance & Extension Flow

The framework is built around a strict layering principle: extend, never modify the base. Two independent inheritance chains exist:

tamsi/ 🔒 → extends → app/     usr_generated/ 🔒 ← generates ← util/
→ extends → usr/
Golden Rule: Never edit tamsi/ or usr_generated/ directly. Always extend via app/ (for core framework) or usr/ (for table-specific logic). This ensures upgrades and regenerations are safe.

Request Lifecycle

  1. User requests a URL, which hits public_html/index.php
  2. index.php forces HTTPS, loads paths config, then instantiates Tamsi (from tamsi/Tamsi.php)
  3. Tamsi (extends Core) creates Model, Controller (with Model dependency), and View
  4. Controller calls receive() to parse the request, prepareMenu() to build navigation
  5. View renders HTML — the controller's displayInclude() resolves the view path and includes it
  6. Output is served back to the browser

Key Source Files

Entry Point — public_html/index.php

<?php if(!isset($_SERVER["HTTPS"]) || $_SERVER["HTTPS"] != "on") { header("Location: https://" . $_SERVER["HTTP_HOST"] . $_SERVER["REQUEST_URI"], true, 301); exit; } session_start(); require_once '../config/paths.cfg.php'; $paths = new Paths(); require_once 'Tamsi.php'; $tamsi = new Tamsi(); // ... renders HTML page with Bootstrap, header, sitemap, content, footer ... $tamsi->view->display(MAIN_CONTAINER);

Main Framework Class — tamsi/Tamsi.php

<?php require_once '__DIR__.'/../config/paths.cfg.php'; require_once 'Core.php'; require_once '../app/view/View.php'; require_once '../app/model/Model.php'; require_once '../app/controller/Controller.php'; $paths = new Paths(); $paths->init(); class Tamsi extends Core { public function __construct(...) { session_start(); $this->model = new Model(); $this->controller = new Controller($this->model); $this->controller->receive(); $this->controller->prepareMenu(); $this->view = new View($this->controller); } }

Application Controller — app/controller/Controller.php

<?php require_once 'TamsiController.php'; class Controller extends TamsiController { public function __construct(Model $model) { parent::__construct($model); } public function displayInclude() { $base_path = $this->model->getBasePath(); $path = $this->model->getPath(); if (!$this->is_allowed_dir($path) && ...) { $_SESSION['tamsi_notice'] = 'Requires login to view!'; return false; } // ... includes the appropriate view file ... } }

Generated Controller — usr_generated/controller/TamsiConfigurationsController.php

<?php require_once 'TamsiSecurity.php'; require_once 'Configuration.php'; require_once 'Next.php'; class TamsiConfigurationsController { public $views; public $security; public function __construct($views) { $this->views = $views; $this->security = new TamsiSecurity(); } protected function validate() { // Validates required fields, email format, etc. } // ... auto-generated CRUD methods ... }

User Extension — usr/controller/ConfigurationsController.php

<?php require_once 'TamsiConfigurationsController.php'; class ConfigurationsController extends TamsiConfigurationsController { // User adds custom logic here without modifying the generated base }

Code Generator — util/generateMVC

Config Overview

File Contents
config/Config.php Aggregator class that loads Paths and Secure_config objects
config/paths.cfg.php Defines Paths class with include paths, URL paths, and the init() method that registers autoloaders and sets PHP configuration
config/database.cfg.php Database connection credentials and DSN
config/main.cfg.php Global constants: GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, FACEBOOK_CLIENT_ID, FACEBOOK_CLIENT_SECRET, PROJECT, MAIN_CONTAINER, and more. Supports machine-specific overrides via main.cfg.php.home
config/security.cfg.php Security settings — CORS headers, allowed origins, content security policies
config/routes.cfg.php URL routing definitions — maps URL paths to controller commands and variables via the Routes class (extends TamsiRoutes)
config/views.cfg.php View rendering options, template paths, layout configuration
